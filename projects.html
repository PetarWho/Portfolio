Projects

Feel free to reach out to me if you are interested in collaborating on my projects or if you would like to invite me to join your own projects. I am open to opportunities and would be delighted to discuss potential partnerships. Please don't hesitate to contact me.

Duplex.gg

I have developed an ASP.NET Core Web Application that integrates Riot's API, Google Drive API, EF Core, and MS SQL. This application serves the purpose of facilitating participation in events organized by the app's administrator. In these events, users are required to complete specific tasks in the game League of Legends. Once a task is completed, users share their game ID with the app, which then queries Riot's API to verify the task completion. Successful completion results in the doubling of entry coins spent to participate in the challenge. These coins can be used to acquire various rewards available within the app. Additionally, the application provides daily free rewards to users based on their rank within the app.

TapVacation (internship)

Our team, consisting of eight individuals, was divided into specialized Backend and Frontend teams. Together, we developed a robust Angular application tailored for both employers and employees. This app facilitates the seamless submission of absence requests, requiring employees to provide detailed explanations and specify the desired duration of their absence. Employers, on the other hand, can efficiently review these requests and make informed decisions by either approving or rejecting them.

Dungeon Runner

During the early stages of my coding journey, I created a Unity game that revolved around exploration and combat. The game featured multiple dungeons where players had to search for four unique weapons. In order to progress, they had to eliminate every enemy and defeat powerful bosses within each level. The game was designed with speedrunning in mind, encouraging players to complete levels as quickly as possible. Unfortunately, due to my lack of experience with Github at the time, the game's code and assets are no longer accessible.

Milk Clicker Universe

I had been developing a clicker game specifically designed for mobile platforms, with the intention of releasing it on various App Stores. Initially, I had an abundance of ideas for the game and set ambitious goals for its development. However, this overwhelming workload ultimately caused me to lose motivation. Therefore, all the game ideas remain documented on Azure DevOps, while the project itself is currently "frozen" and kept privately stored on Github.

SpeedTyper

I have been developing an engaging word-falling game, challenging players to type every descending word correctly while incurring penalties for mistakes. The game offers four main difficulty levels, catering to various skill levels, and also includes additional modes for players to enjoy. By collecting coins and trophies, players can enhance their typing speed and accuracy in an enjoyable and interactive manner. I had originally planned to release this game on Steam later this year.

LeagueChamps

I developed a Windows Forms application using C# that serves as a tool to track my unowned champions in a specific game. With this app, I can quickly determine the total number of champions I don't possess and calculate the amount of in-game currency (BE) required to acquire them all. The only drawback is that I need to manually enter the relevant information. However, I have plans to enhance the application by integrating it with Riot's API, allowing for automated retrieval of champion data. You can find the current version of the application on my GitHub repository.

Play it Wright

I've developed an automated solution using Playwright, an end-to-end testing package, to streamline the process of claiming the free game offered by Epic Games on a weekly basis. By leveraging Playwright's capabilities, I have created a script that handles the entire process, from logging in to claiming the free games, for a hundred different accounts simultaneously. However, the only challenge that persists is the Captcha mechanism, which presents an obstacle to complete automation. I continue to explore ways to overcome this hurdle and improve the efficiency of the process.
